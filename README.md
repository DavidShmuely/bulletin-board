# לוח מודעות דיגיטלי — מערכת רב-לקוחות

מנוע אחד (`index.html`) שמשרת כל מספר של בניינים. כל בניין = Spreadsheet אחד, וכל ההגדרות — layout, צבעים, תוכן — נשלטות משם.

```
GitHub Pages (index.html — מנוע אחד)
        │  ?client=tamar
        ▼
Apps Script אחד ── master: גיליון clients (מזהה → Spreadsheet ID)
        ▼
Spreadsheet לכל בניין ── תיקיית Drive לכל בניין (תמונות)
```

---

## הקמה חד-פעמית (כ-20 דקות)

### שלב 1 — Master Spreadsheet
1. צור Google Spreadsheet חדש בשם `bulletin-master` (שלך בלבד — לא משותף לוועדים).
2. ייבא את `master_template.xlsx` (קובץ ← ייבוא ← החלפת גיליון).
3. שמור את ה-ID של ה-Spreadsheet (החלק הארוך ב-URL בין `/d/` ל-`/edit`).

### שלב 2 — Apps Script
1. ב-[script.google.com](https://script.google.com) צור פרויקט חדש (standalone).
2. הדבק את התוכן של `apps_script.gs`.
3. בשורה הראשונה, החלף את `PASTE_MASTER_SPREADSHEET_ID_HERE` ב-ID מהשלב הקודם.
4. פרסום: **Deploy → New deployment → Web app**
   - Execute as: **Me**
   - Who has access: **Anyone**
5. שמור את כתובת ה-Web App (`https://script.google.com/macros/s/.../exec`).
6. בהרצה הראשונה — אשר הרשאות (Sheets + Drive).

### שלב 3 — המנוע
1. ב-`index.html`, החלף את `PASTE_APPS_SCRIPT_URL_HERE` בכתובת מהשלב הקודם.
2. צור repo חדש ב-GitHub (ציבורי), העלה את כל תוכן התיקייה.
3. **Settings → Pages → Source: main branch** → שמור.
4. הכתובת: `https://USERNAME.github.io/REPO/`

---

## הוספת בניין חדש (כ-5 דקות)

1. **Spreadsheet:** ייבא את `client_template.xlsx` ל-Google Sheets חדש בשם הבניין. ערוך את `settings` (שם, מיקום, צבעים).
2. **תיקיית תמונות:** צור תיקיית Drive, שתף כ-**"כל מי שיש לו קישור — צפייה"**, והדבק את ה-ID שלה במפתח `תיקיית_תמונות`.
3. **רישום:** הוסף שורה ב-master ← `clients`: מזהה באנגלית (למשל `erez`), ה-Spreadsheet ID, `כן`.
4. **שיתוף לוועד:** שתף את ה-Spreadsheet של הבניין (עורך) + את תיקיית התמונות עם חברי הוועד בלבד.
5. **כתובת נקייה (רשות):** שכפל את התיקייה `tamar/` בשם המזהה החדש ועדכן בתוכה את שם ה-client.
6. **טלוויזיה:** התקן Fully Kiosk Browser והגדר Start URL:
   `https://USERNAME.github.io/REPO/?client=erez`

> ⚠️ אין צורך ב-deploy חדש של Apps Script בהוספת בניין — רק שורה בגיליון clients.

## מדריך ההגדרות (גיליון settings)

### Layout — שלושה מפתחות
| מפתח | דוגמה | משמעות |
|---|---|---|
| רשת_עמודות | `1, 2.5, 1` | רוחב יחסי של כל עמודה |
| רשת_שורות | `1.4, 1` | גובה יחסי של כל שורה |
| מפת_אזורים | `הודעות קרוסלה זמנים / הודעות מזג_אוויר שבת` | מה מוצג בכל תא; `/` מפריד שורות; `.` = תא ריק |

**רכיבים זמינים:** `הודעות`, `קרוסלה`, `זמנים`, `שבת`, `מזג_אוויר`

רכיב שמופיע בכמה תאים סמוכים — משתרע עליהם. דוגמאות:

```
# חלון ענק במרכז, הודעות לכל הגובה
רשת_עמודות:  1, 3, 1
רשת_שורות:   1, 1
מפת_אזורים:  הודעות קרוסלה זמנים / הודעות קרוסלה שבת

# 6 חלונות שווים
רשת_עמודות:  1, 1, 1
רשת_שורות:   1, 1
מפת_אזורים:  הודעות קרוסלה זמנים / מזג_אוויר קרוסלה שבת

# שתי עמודות בלבד, בלי מזג אוויר
רשת_עמודות:  1, 2
רשת_שורות:   1
מפת_אזורים:  זמנים קרוסלה
```

### תמונות (קרוסלה)
- **עדיפות 1:** גיליון `central` — רשימת URLים ידניים.
- **עדיפות 2:** אם `central` ריק — כל התמונות מתיקיית ה-Drive (`תיקיית_תמונות`), ממוינות לפי שם קובץ (`01_...`, `02_...`).
- `מרווח_תמונות`: מספר = שניות בין תמונות; `08:00` = החלפה יומית בשעה קבועה.
- התמונות נטענות כ-thumbnail ישירות מ-Drive — אין מגבלת כמות.

### גלילה אוטומטית
כשההודעות לא נכנסות בחלון — הן נגללות בלולאה רציפה אוטומטית. `מהירות_גלילה` שולט בקצב (פיקסלים/שנייה). כשהכל נכנס — אין תזוזה.

### תוכן לוועד
- **notices** — אייקון (אימוג'י), כותרת, טקסט (שורה חדשה עם Alt+Enter).
- **footer** — הודעות מתחלפות בתחתית המסך.

## תחזוקה

- **עדכון קוד:** push ל-GitHub — כל הלוחות מקבלים את הגרסה החדשה ברענון הבא.
- **עדכון Apps Script:** אחרי שינוי קוד — **Deploy → Manage deployments → Edit → New version** (אחרת השינוי לא עולה!).
- **תמונות Drive:** נשמרות ב-cache של 5 דקות בצד השרת — תמונה חדשה תופיע תוך ~5 דקות + רענון נתונים.
- **לוגו קבוע:** העלה ל-`assets/BUILDING/logo.png` ב-repo וכתוב את הנתיב במפתח `לוגו`.
